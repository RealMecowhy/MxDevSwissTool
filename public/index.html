How to configure Mendix Telemetry & Metrics

💻 A. Setting up the Local Agent Mode (Recommended for Local Dev)

The Local Observability Agent connects directly to PostgreSQL and watches your Mendix runtime logs. To use it:

  1. Find the file mendix-observability-bridge.js inside your project directory.
  2. Open your terminal and run the command: 
    node mendix-observability-bridge.js
  3. In the Swiss Tool telemetry dashboard toolbar, select Local Agent Mode and click Connect Agent.
  4. It will auto-detect your PostgreSQL settings and tail the application logs! Live logs will stream directly to the Log Viewer tab of this toolkit.

🚀 OpenTelemetry (Mendix 11.12+)

The Local Observability Agent includes a built-in OpenTelemetry (OTLP) Collector that listens on port 4318 for Traces and Logs. It natively supports both the default binary **Protobuf** format (`http/protobuf`) and **JSON** (`http/json`).

  1. In Mendix Studio Pro, go to App Settings → Configurations → Edit.
  2. In the OpenTelemetry tab, check Enable OpenTelemetry.
  3. Set Endpoint to: http://127.0.0.1:4318 (using IP resolves local DNS issues on Windows).
  4. Set Enable Traces and Enable Logs to Yes.
  5. Note: You do **not** need to add any Extra JVM parameters! (We recommend removing `-Dotel.exporter.otlp.protocol=http/json` if you added it earlier, as Mendix's JSON exporter can be unstable locally).
  6. Run your app and ensure the Local Agent is connected. Traces will appear live in the "OTLP Traces" tab!

1. Enable Prometheus Metrics Registry

To enable metric exporter in Mendix Studio Pro → Settings → Configurations (Edit) → Custom:

Custom Setting Key:
Metrics.Registries
Value (JSON string):
[
  {
    "type": "prometheus",
    "settings": {
      "step": "10s"
    }
  }
]

2. Add App Tags for Filtering (Optional)

Custom Setting Key:
Metrics.ApplicationTags
Value (JSON string):
{
  "environment": "local-dev",
  "application": "SwissToolDemo"
}
